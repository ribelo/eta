# Restructure to packages/ monorepo layout

## description

The repo currently keeps effet's source at lib/ and tests at test/ directly under the root. To add a sibling effet-otel package and to match the reference effect-smol layout (and Rust workspace conventions), source should live under packages/effet/ with its tests at packages/effet/test/. This restructure must happen before either the observability work or the effet-otel package is built, because both will land in the new layout.

## design

Move lib/ -> packages/effet/ and test/ -> packages/effet/test/. dune-project flips to (generate_opam_files true) and declares the effet package with synopsis and dependencies. Delete the legacy apsis.opam file; dune will regenerate effet.opam at the repo root. flake.nix and flake.lock stay at the repo root. scratch/ stays at the repo root. Update any dune files inside packages/effet/ for the new internal paths. Update README.md path references if any. Verify journal references remain valid (journal.md is gitignored so it can be touched freely).

## acceptance criteria

Repository layout has packages/effet/ containing the effet library source, packages/effet/test/ containing the test executable, and dune-project at the repo root declaring the effet package. apsis.opam is removed and effet.opam exists at the repo root, generated by dune. nix develop -c dune build succeeds with no source changes beyond the move. nix develop -c dune runtest --force passes all 45 existing tests. The README.md still describes building and testing accurately.

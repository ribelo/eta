# P2: In-memory tracer must emit valid span_id and trace_id

## description

Bug: packages/eta/tracer.ml inspect (line 209) returns Capabilities.span_info with span_id = '' (empty string). Root spans default to trace_id = '' when there is no parent and no external_parent (line 120). Effect.current_span and Effect.current_context propagate these empty values. This weakens log correlation, downstream propagation, and any code that uses the in-memory tracer as a local backend.

Location: packages/eta/tracer.ml inspect (line 209), root-span trace_id default (line 120)

## design

RED test (write first):
1. Build a runtime with Tracer.in_memory ().
2. Run an effect Effect.named ~kind:Internal 'root' (Effect.bind Effect.current_span (fun info -> ...)).
3. Assert returned span_info.span_id is a 16-hex string (length 16, all 0-9a-f).
4. Assert info.trace_id is a 32-hex string.
5. Currently both are empty strings.

Regression checks:
- A child span inherits parent trace_id.
- Externally-provided trace_context (With_external_parent) overrides trace_id correctly.

Fix shape:
- Generate W3C-shaped 16-hex span_id per span and 32-hex trace_id for root spans, using the runtime's Capabilities.random.
- Keep the integer span_id as an internal handle only; never expose it as the propagation span_id.
- Update inspect to return the W3C span_id.

## acceptance criteria

RED test fails on current code and passes after the fix. Both regression checks hold. The integer handle remains for internal correlation only.

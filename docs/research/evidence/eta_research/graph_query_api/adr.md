# Graph Query API ADR

Status: accepted for implementation planning
Date: 2026-05-27
Evidence root: scratch/eta_research/graph_query_api/

## Decision

Build lib/graph/'s query surface as a layered API:

- Baseline: parameterized Cypher string plus typed decoder.
- Typed builder: hybrid Pattern DSL plus pipeable clause builder for common read queries.
- Escape hatch: explicit raw Cypher fragments/clauses, not hidden fallbacks.
- PPX: schema PPX for Node modules first; no literal Cypher PPX as primary v0.1 surface.

## Public Shape

### Graph.Query

- query : Connection.t -> cypher:string -> params:Param.t list -> decode:'a Decode.t -> ('a list, error) Effect.t
- exec : Connection.t -> cypher:string -> params:Param.t list -> (unit, error) Effect.t
- prepare : cypher:string -> prepared
- execute : Connection.t -> prepared -> params:Param.t list -> decode:'a Decode.t -> ('a list, error) Effect.t

This is Branch D and remains the guaranteed surface.

### Graph.Param

- string : string -> string -> t
- int : string -> int64 -> t
- float : string -> float -> t
- bool : string -> bool -> t
- null : string -> t
- list : string -> ('a -> value) -> 'a list -> t
- map : string -> field list -> t

Bytes stays absent or explicitly Unsupported until LadybugDB exposes a blob bind/create path.

### Graph.Decode

Primary shape:

- node : 'schema node_schema -> 'schema Decode.t
- string : string Decode.t
- int : int64 Decode.t
- tuple2 / tuple3 for heterogeneous rows
- record decoders generated by schema PPX as optional sugar

Do not use an hlist/applicative decoder as the primary surface.

### Graph.Pattern and Graph.Builder

Typed ergonomic layer for the tested subset:

- Pattern.node
- Pattern.rel
- Pattern.path
- Pattern.anon_node
- rel direction and hop range
- Query.match_
- Query.optional
- Query.where
- Query.with_
- Query.return
- Query.order_by
- Query.limit
- Query.unwind / merge / set only if implementation can keep them small; otherwise use raw exec for writes

The pattern DSL is graph-shaped; the clause builder is pipe-shaped. Do not try to make the pattern itself a SQL-style pipe.

### Graph.Fragment

Optional app-level helper layer for Branch E:

- named reusable pattern fragments
- raw_expr and raw_clause are explicit in the type/name
- intended for stable domain graph motifs and bulk ingest recipes

## PPX Role

Accepted:

- schema PPX for NODE modules that generate label metadata, typed property accessors, parameter constructors, and NODE decoders for the Arrow shape proven by LadybugDB P-Lbug-1.

Partial:

- REL PPX metadata and pattern constructors.

Deferred:

- REL/PATH decoders until Arrow decoding is proven.
- Cypher literal PPX beyond parameter/schema lint.
- Return-shape inference PPX, because it requires a real Cypher parser or LadybugDB metadata.

## Escape Hatch Boundary

Raw Cypher is not a hidden compatibility shim. It is a named, first-class baseline and escape hatch:

- query ~cypher
- exec ~cypher
- raw_expr
- raw_clause
- raw_write

The builder must not silently fall back to strings. If a branch requires raw Cypher, the call-site should say so.

## Known Gaps

- Compile-time parameter type checking is not proven.
- Literal PPX return inference is not proven.
- REL/PATH decoding is not proven.
- Bytes parameters are blocked.
- The builder write surface is not proven beyond workload Q9's shape.

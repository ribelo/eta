# A1 provider diff probe

Question: should eta-ai compose providers as values with data and a few small
provider functions, or should each provider be a separate module shape?

Assumptions:

- OpenAI-compatible is a compatibility profile, not one single provider. It
  means the OpenAI chat-completions envelope with a configurable base URL and
  bearer token.
- The probe covers chat completions/messages, streaming, tool calling, and
  error JSON shape. Provider-specific model catalogs and pricing are out of
  scope for A1.
- TypeScript Effect AI reference code can suggest comparison concerns, but the
  verdict is based on provider docs and Eta constraints.

Proof obligations:

- Identify which differences are pure data: base URL, path, auth headers,
  optional headers, and provider routing flags.
- Identify which differences are structural: request envelope, response
  envelope, streaming state machine, tool-call representation, and error JSON.
- Decide whether the structural differences are still small enough for a
  provider record with encoder/decoder function slots.
- Trigger the disproof if streaming is structurally different across all four
  provider columns in a way that cannot be represented by a tagged stream event
  or a per-provider stream decoder.

Candidate ledger:

| Candidate | Why plausible | Evidence needed to win | Falsifier | Status |
| --- | --- | --- | --- | --- |
| Providers as data only | Most provider docs expose URL, headers, model, messages, tools, stream, and errors as JSON fields. | Matrix shows differences are identical or parametrized by data. | Anthropic request/response/stream shapes require behavior, not just field values. | Rejected for A1. |
| Provider value with small functions | Eta can keep one core API while provider values own encode/decode details. | Structural cells are localized to request, response, stream, and error codecs. | A provider needs a different public control flow or streaming ownership model. | Accepted for A1. |
| Per-provider modules | OCaml modules naturally isolate very different APIs. | Structural differences dominate and cannot be expressed by a common tagged domain model. | Most differences are data or localized codecs; modules add surface before evidence requires them. | Not selected for A1. |

Run:

    bash scratch/eta_ai_v1/probes/provider_diff/run.sh

Outputs:

- provider_matrix.md records the provider diff.
- verdict.md records the A1 decision and disproof outcome.
- results/doc-checks.txt is generated by run.sh and records source
  reachability checks.

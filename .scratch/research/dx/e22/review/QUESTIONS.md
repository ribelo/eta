# Maintainer review questions

Review `LAWS.md` as a model document, not merely as a test checklist.

1. Does every mli-stated row cite the exact normative sentence it claims? Is any
   behavioral equation/order/lifecycle statement incorrectly presented as a
   model law, or vice versa?
2. Is `Exit.t` plus ordered `Eta_test.Run.event` under a seeded fresh runtime the
   right observation equivalence? Which event should be intentionally hidden or
   added?
3. Are total enumerated bind functions and finite deterministic blueprints an
   honest generated class? Which excluded function class deserves a named
   adversarial property?
4. Are fail-fast and race correctly stated at combinator completion—after
   cancellation-protected cleanup—without claiming a scheduler deadline?
5. Does each matrix property directly discriminate every one-claim census row
   that cites it, or is a repeated property name hiding an unasserted claim?
6. Should the two schedule model laws be promoted into normative prose now? If
   so, what exact constructor domains make monotonicity and recurrence count
   public promises rather than implementation observations?
7. Do the separate clock/random/logger/tracer rows have exact restoration,
   innermost-wins, and sibling-isolation evidence rather than inference from a
   shared implementation?
8. Do the primitive trace generators cover the meaningful fence and
   cancellation states without implying scheduler fairness?
9. Can any property pass after deleting an equation side, truncating a generated
   prefix, avoiding the documented exit/branch, leaking a fiber, or serializing
   an ordering test? If so, reject it as vacuous.
10. Are any corrected boundaries missing from the public mli prose? A prose bug
    needs its counterexample recorded before it is fixed.
11. For every R-row, does the exact displayed test name exist and directly
    discriminate the cited claim, or is the registry inferring coverage from a
    nearby suite? Reject helper-only and unnamed pointers.
12. Is the scope statement honest: five census-complete modules today,
    repository-wide same-change enforcement prospectively, and dated/owned
    retrospective debt elsewhere—without wording that implies all historical
    public interfaces are already covered?
